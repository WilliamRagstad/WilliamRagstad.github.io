Grammar

The first part of the compilation process is the lexical analysis stage where characters are transformed into tokens. After this part, all tokens are parsed into what is called a abstract syntax tree (AST). This is exactly what the grammar is specifying, what rules the lexer and parser should follow. For this, we use a PEG (parsing expression grammar) parser library called pegjs.
Read more about the Grammar Syntax and Semantics documentation used by pegjs.

Code generation

To specify a code generation stage is optional, but keep in mind that the compiler function will be named 'interpret()' instead of 'compile()'.
The code generator is located in the second column in the first row in the editor. The first function in the code must be the entry point, meaning the first function is always called by the built compiler.
